# 需求变更说明

## 目标

将当前基于 `Makefile` 的启动方式改造成基于 Bun 的工程，并输出 macOS 平台可执行二进制。

## 变更要求

- 移除 `Makefile` 作为主入口，后续只保留 Bun 工程链路
- 使用 Bun 构建并产出可执行文件
- 内置 `scenes/` 中的场景文件，不再依赖运行时读取仓库文件
- Claude 保留流式输出能力；Codex 下 `jj -s` 与 `jj -p` 行为一致

## 兼容要求

- 对外只保留一个全局命令 `jj`
- 非交互能力通过 `jj -p`、`jj -s` 提供
- 使用者不需要再手动配置 shell function
- 现有场景短名与 `.code`、`.d` 这类 Codex 前缀写法必须继续可用
- `-p` / `-s` 启动后统一在终端提示符中输入内容
- 不再支持通过命令行参数、pipe、`<<<` 或 Heredoc 传入内容

## 验收标准

- 可以通过 Bun 完成安装、构建
- 可以生成 macOS `arm64` 和 `x64` 二进制
- 用户只需要使用 `jj [scene]`、`jj -p [scene]`、`jj -s [scene]`
